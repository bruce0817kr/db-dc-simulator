# DB/DC 시뮬레이터 디자인 시스템

## 1. 제품 인상

- 금융 판단을 돕는 차분하고 보수적인 도구다.
- 장식보다 정보 위계, 읽기 쉬운 숫자, 명시적인 위험 안내를 우선한다.
- 투자 권유나 확정 수익처럼 보이는 과도한 강조를 사용하지 않는다.

## 2. 색상

- 기본 배경: 흰색과 회색 계열(`white`, `gray-50`~`gray-900`).
- 주요 행동: 파란색 계열(`blue-500`~`blue-700`).
- 고급 설정과 주의: 호박색 계열(`amber-50`, `amber-200`, `amber-700`, `amber-800`).
- 오류: 빨간색 계열(`red-400`~`red-600`).
- 성공 메시지는 별도 장식색을 추가하지 않고 본문 회색으로 전달한다.

## 3. 타이포그래피

- 본문은 시스템 산세리프를 사용한다.
- 본문 입력과 버튼은 `text-sm`, 보조 설명과 검증 메시지는 `text-xs`를 기본으로 한다.
- 섹션 제목은 `text-lg font-semibold`, 필드 라벨은 `text-sm font-medium`을 사용한다.
- 금액과 수익률은 꾸밈보다 단위와 자릿수 구분을 명확히 표시한다.

## 4. 간격과 레이아웃

- 4px 배수의 Tailwind 간격 체계를 사용한다.
- 기본 폼은 `gap-4`, 밀접한 안내와 상태는 `gap-1` 또는 `gap-2`를 사용한다.
- 페이지는 모바일 단일 열, 큰 화면에서 입력 400px과 결과 유동 열로 구성한다.
- 터치 대상 최소 높이는 44px이다.

## 5. 재사용 프리미티브와 상태

- `Button`: primary/secondary, hover, focus-visible, disabled 상태를 제공한다.
- `TextInput`: label, suffix, placeholder, focus, invalid, error message 상태를 제공한다.
- `Select`: label, focus, invalid, error message 상태를 제공한다.
- `Card`: 흰 배경, 회색 테두리, 작은 그림자로 결과 그룹을 구분한다.
- `details/summary`: 고급 기능을 기본 흐름에서 접어 두는 점진적 공개 패턴이다.
- 공유 옵트인: 네이티브 checkbox와 연결된 label을 사용하며, 선택 전후 설명이 민감정보 포함 여부를 명확히 말해야 한다.

## 6. 상호작용과 접근성

- 상태 변화는 색상만으로 전달하지 않고 한국어 문구를 함께 제공한다.
- 모든 입력은 연결된 label을 가지며 오류는 `aria-invalid`와 설명 ID로 연결한다.
- 키보드 focus ring을 유지하고, 비활성 버튼은 포인터와 키보드 입력을 차단한다.
- 공유 성공/실패 메시지는 사용자가 복사 결과를 확인할 수 있게 텍스트로 표시한다.

## 7. 반응형과 인쇄

- 375px, 768px, 1280px에서 가로 스크롤 없이 동작해야 한다.
- 긴 연도별 입력은 제한된 높이 안에서 세로 스크롤한다.
- 입력, 공유 제어 등 조작 UI는 인쇄에서 숨기고 결과 보고서만 유지한다.

## 8. 허용된 디자인 부채

- 현재 전역 폰트와 일부 색상은 Tailwind 유틸리티에 직접 표현되어 있다.
- 전용 컴포넌트 쇼케이스는 없으며 Vitest 상태 테스트와 Playwright 반응형 QA를 동등한 상태 하네스로 사용한다.
- PR 17은 기존 시각 언어를 보존하며 새 색상·의존성·애니메이션을 도입하지 않는다.
