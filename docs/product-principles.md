# 제품 원칙 (Product Principles)

본 문서는 DB/DC 퇴직연금 전환 시뮬레이터의 제품 철학과 표현 규칙을 정의한다.
코드, UI 문구, 문서 작성 시 본 원칙을 우선한다.

---

## 1. 서비스 정체성

- 이 서비스는 **시뮬레이션 도구**이며, 투자 권유 도구가 아니다.
- 사용자가 자신의 임금상승률(확정적 복리)과 DC 운용수익률(불확실한 복리)을 비교하도록 돕는 것이 핵심 목적이다.
- 특정 금융상품 가입, 매수, 전환을 안내하거나 권하지 않는다.

## 2. 투자 권유 금지 원칙

금지 표현:
- "S&P 500 ETF에 가입하세요"
- "NASDAQ 100이 더 유리합니다"
- "이 상품을 사면 됩니다"
- "추천", "가입", "매수" 등 권유·지시 어휘

권장 표현:
- "S&P 500 지수 추종형 자산을 편입한다고 가정합니다"
- "NASDAQ 100 지수 추종형 자산의 기대수익률을 사용자가 입력한 경우입니다"
- "이 결과는 상품 추천이 아니라 시뮬레이션입니다"

## 3. 불확실성 표현 원칙

DC 운용수익률은 입력값에 따른 **시뮬레이션 가정**이며 확정 수익이 아니다.

필수 표현:
- "본 결과는 입력한 가정에 따른 시뮬레이션입니다"
- "실제 결과는 임금 변동, 회사 퇴직연금 규약, 운용성과, 수수료, 세금, 제도 변경에 따라 달라질 수 있습니다"
- 리스크 섹션에는 "확정 예측이 아닙니다" 명시

금지 표현:
- "확정 수익", "보장 수익률", "무조건 유리", "반드시 이긴다"

## 4. 단위 규약

- 금액: KRW, JavaScript `number` 타입 (원 단위)
- 수익률·상승률: decimal rate (`0.03` = 3%)
- 연수: 정수 (`number` 타입, 정수만 유효)
- UI 표시: `3%`처럼 퍼센트로 표시하되, 계산 엔진에는 `0.03` decimal로 전달
- UI 표시와 계산 단위를 분리한다. 혼동 시 계산 오류·문구 오류의 원인이 된다.

## 5. 계산/UI 분리 원칙

- 계산 로직은 `src/calculator/` (deterministic pure TypeScript function)
- UI는 `src/features/simulator/` (컴포넌트 / 훅 / 유틸 / 타입 분리)
- 공통 UI는 `src/components/ui/`
- UI 컴포넌트 안에 계산식을 중복 구현하지 않는다.
- 계산 함수는 `Date`, `locale`, browser API에 의존하지 않는다.
- 동일한 input은 항상 동일한 output을 반환한다.

## 6. 프리셋 표현 규칙 (S&P 500, NASDAQ 100)

- S&P 500, NASDAQ 100은 특정 ETF 종목 추천이 아니라 **지수 추종형 자산을 가정**하는 방식으로만 표현한다.
- 프리셋의 기대수익률·변동성·보수는 서비스의 가정치이며 시장 예측이나 수익 보장이 아니다.
- 프리셋 설명에는 "지수 추종형 자산을 편입한다고 가정합니다" 표현을 사용한다.
- 프리셋 안내에는 "가정치이며 예측이나 보장이 아닙니다" 문구를 포함한다.

## 7. 결과 제공 원칙

- 결과는 숫자만 보여주지 않고 **숫자 + 문장**으로 제공한다.
- 핵심 문장: "입력하신 조건에서는 DC 전환 후 연평균 X.X% 이상 운용해야 DB 유지보다 유리합니다."
- 차이 금액 표시 시 DC 유리(양수) / DB 유리(음수) / 거의 동일(TIE)을 텍스트 라벨로 함께 표시한다.
- 색상에만 의존하지 않는다. DB/DC 배지는 색과 텍스트 라벨을 동시에 제공한다.

## 8. 세금 표현 원칙

- 기본 결과는 **세전 기준**이다.
- 세후 추정은 옵션 기능이며, 2025년 세법 기준 추정치임을 명시한다.
- 세금 계산은 서비스 공개 전 세무 전문가 검증이 필요한 영역이다.
- "세전 시뮬레이션" 표현을 기본 결과에 명시한다.

## 9. 데이터 처리 원칙

- 사용자 입력(연봉, 근속연수 등)은 브라우저 안에서만 처리한다.
- 서버 저장, 로그인, 외부 전송을 하지 않는다.
- URL 공유 시 재무 정보가 포함됨을 명시적으로 안내한다.
- 공유 링크에는 연봉 등 민감 정보가 그대로 포함된다.

## 10. 제도 가정 처리 원칙

- 위험자산 한도, DC 부담금 비율 등 제도 가정은 `src/calculator/rules.ts`의 `DEFAULT_RULE_SET`으로 중앙 관리한다.
- 제도 변경 가능성이 있으므로 UI에 "현재 서비스의 계산 가정"으로 표시한다.
- 법령·규정 데이터는 자동 업데이트하지 않는다.
- 향후 관리자가 룰셋을 교체할 수 있는 구조를 전제로 설계한다.
