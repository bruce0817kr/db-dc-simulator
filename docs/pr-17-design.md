# PR 17 설계 — URL 공유 고급 설정 옵트인

## 목표

기본 공유 링크의 최소 노출 원칙을 유지하면서, 사용자가 명시적으로 선택한 경우에만 고급 임금 시나리오를 링크에 포함하고 동일한 상태로 복원한다.

## 정책

- 기본값은 미포함이다. 기존 공유 URL과 동작을 유지한다.
- 고급 임금 경로 또는 평균임금 직접 입력이 활성화된 경우에만 옵트인 체크박스를 표시한다.
- 체크박스는 공유할 때마다 사용자가 직접 선택해야 하며 폼 초기값이나 URL에서 자동 선택하지 않는다.
- `advanced=1`이 있는 링크만 고급 파라미터를 복원한다.
- 모드별로 실제 계산에 필요한 값만 직렬화한다.
- 서버 저장, 외부 API, 암호화 또는 단축 URL은 추가하지 않는다.

## URL 계약

공통 플래그와 모드:

- `advanced=1`
- `salaryMode=CONSTANT_GROWTH|WAGE_PEAK|STEP_UP|YEARLY_CUSTOM`
- `dbAverageSalary=<KRW 정수>`: 값이 있을 때만 포함

모드별 파라미터:

- `WAGE_PEAK`: `peakStart`, `peakCut`, `peakPostGrowth`
- `STEP_UP`: `stepUpYear`, `stepUpRate`
- `YEARLY_CUSTOM`: `salaries=<KRW 정수의 쉼표 구분 목록>`

## 개인정보와 실패 처리

- 체크박스 라벨과 설명에서 연도별 연봉·평균임금이 링크에 노출될 수 있음을 명시한다.
- 파싱할 수 없는 고급 값은 해당 필드에서 제외한다.
- `advanced=1`이 없으면 고급 파라미터가 있어도 무시해 의도하지 않은 복원을 막는다.

## 검증

- 순수 URL 함수의 기본 미포함, 옵트인 포함, 모드별 왕복, 악성·무효 파라미터 테스트
- 공유 컴포넌트의 체크박스 노출과 기본 미선택 테스트
- Playwright에서 기본 미포함과 옵트인 후 새 페이지 복원 테스트
- 375px, 768px, 1280px에서 공유 영역과 체크박스 접근성 확인
